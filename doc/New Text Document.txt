CloudFlix – Netflix-Style Video Streaming Platform on AWS
1. Project Overview

CloudFlix is a scalable Netflix-style video streaming platform built using Microservices Architecture and deployed on AWS Cloud Infrastructure.

The system allows users to:

Upload videos

Receive notifications

Track watch history

This project demonstrates enterprise-level AWS architecture similar to platforms like Netflix.

# 🚀 PHASE 1: VPC SETUP (VERY IMPORTANT)

Goal:

👉 Create **3 isolated VPCs** with proper subnet structure
👉 This is the base of your entire system security

---

# 🧠 Step 1: Decide CIDR Blocks (CRITICAL)

You must **plan IP ranges properly** (no overlap)

### Use this:

```text
Public VPC  → 10.0.0.0/16
App VPC     → 10.1.0.0/16
Data VPC    → 10.2.0.0/16
```

👉 Why?

* Easy to remember
* No conflicts
* Clean routing later

---

# 🏗️ Step 2: Create 3 VPCs

Go to:

👉 **Amazon VPC**

Create:

---

## 🔵 VPC 1: Public VPC

```text
Name: public-vpc
CIDR: 10.0.0.0/16
```

---

## 🟢 VPC 2: App VPC

```text
Name: app-vpc
CIDR: 10.1.0.0/16
```

---

## 🟣 VPC 3: Data VPC

```text
Name: data-vpc
CIDR: 10.2.0.0/16
```

---

# 🌐 Step 3: Create Subnets (VERY IMPORTANT)

We divide each VPC into:

👉 Public Subnet
👉 Private Subnet

---

## 🔵 Public VPC Subnets

| Type           | CIDR        | Purpose       |
| -------------- | ----------- | ------------- |
| Public Subnet  | 10.0.1.0/24 | Load Balancer |
| Private Subnet | 10.0.2.0/24 | API Gateway   |

---

## 🟢 App VPC Subnets

| Type             | CIDR        | Purpose       |
| ---------------- | ----------- | ------------- |
| Private Subnet 1 | 10.1.1.0/24 | Auth Service  |
| Private Subnet 2 | 10.1.2.0/24 | Video Service |

👉 No public subnet here ❌

---

## 🟣 Data VPC Subnets

| Type             | CIDR        | Purpose                        |
| ---------------- | ----------- | ------------------------------ |
| Private Subnet 1 | 10.2.1.0/24 | MySQL / RDS                    |
| Private Subnet 2 | 10.2.2.0/24 | DynamoDB access / future Redis |

👉 Fully private ❌

---

# 🌍 Step 4: Internet Gateway (ONLY FOR PUBLIC VPC)

Create:

👉 Internet Gateway

Attach it to:

```text
public-vpc ONLY
```

---

# 🛣️ Step 5: Route Tables

---

## 🔵 Public Subnet Route Table

```text
Destination     Target
0.0.0.0/0  →    Internet Gateway
```

👉 This allows internet access

---

## 🟢 App VPC Route Table

```text
No internet route ❌
```

---

## 🟣 Data VPC Route Table

```text
No internet route ❌
```

---

# ⚠️ Step 6: Enable Auto-Assign Public IP

ONLY for:

👉 Public subnet in Public VPC

Enable:

```text
Auto-assign public IPv4 → YES
```

Everything else:

```text
NO ❌
```

---

# 🔐 Step 7: Basic Security Groups (Initial)

Create simple SGs (we’ll refine later):

---

## Public SG

```text
Allow:
HTTP (80)
HTTPS (443)
SSH (22) → your IP only
```

---

## App SG

```text
Allow:
Port 3000 (Node.js)
From → Public VPC CIDR (10.0.0.0/16)
```

---

## Data SG

```text
Allow:
MySQL (3306)
From → App VPC CIDR (10.1.0.0/16)
```

# 🚫 Common Mistakes (Avoid This)

❌ Overlapping CIDR
❌ Making DB public
❌ Giving internet to App/Data VPC
❌ Using same subnet for everything
